Cluster I — the coverage-snap that silently stopped snapping

When a chat query lands on a date range that has no data, the service is meant to probe the cube's real coverage and re-run against the latest window that does have data — so the card shows a chart, not an empty box. Three high-severity bugs quietly defeated that, and a single pinned date got dragged around when it shouldn't be.
