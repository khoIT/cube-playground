# mf_users pre-agg coverage optimization

Cut the Trino load generated by `mf_users` queries that bypass CubeStore rollups. **Re-scoped
after phase-00** (see `plans/reports/scout-260619-1544-mf-users-heavy-query-attribution-report.md`):
phase-00 disproved the original premise — mf_users queries never scan `etl_ingame_moneyflow`,
and the composition rollup (P2) is already built and working. Remaining mf_users work is a
narrow pre-agg-coverage gap.

## Evidence (CORRECTED by phase-00 — verified via file forensics + live compiled SQL)

- **mf_users never touches `etl_ingame_moneyflow`.** Base SQL = `mf_users base LEFT JOIN
  (mf_ingame_roles GROUP BY)`; all 5 probed shapes compile to `FROM mf_users, mf_ingame_roles`.
  No member query pulls an event-table measure (emitter file:line in the phase-00 report).
- **P2 ALREADY DONE + WORKING.** `user_composition_batch` (commits `909d80df`, `2c7c4cbd`)
  serves payer_tier/lifecycle_stage/is_paying_user/country/media/os with approx counts +
  additive LTV. Live probe → `usedPreAggregations:[mf_users.user_composition_batch]`,
  `external:true`. COHORT rollup also HIT.
- **Real remaining MISS (verified today):** (1) `churn_risk` / `engagement_segment` aggregates
  + `whales` segment + exact `user_count` — no rollup covers those dims/segment. (2)
  member-detail projection + identity-size re-run the live snapshot view (full table scan +
  `mf_ingame_roles GROUP BY`) every time.
- **The cited ≥5 min `etl_ingame_moneyflow` queries belong to a DIFFERENT cube** —
  `etl_money_flow` (1.35B rows), consumed by economy/gacha dashboards + diamond
  business-metrics. Out of scope here (separate future plan).

## Outcome

- Aggregate MISS shapes (churn_risk / engagement_segment group-by + whales/at_risk-scoped cards) now serve from CubeStore on all 8 games: ~10–21s raw Trino → ~0.01s.
- No regression on existing composition/cohort rollups. ptg has 0 payers (all non_payer) — correct, not a bug.
- **Deploy note:** changing a rollup def mints a new version; queries route to it immediately but ERROR until the worker rebuilds (prod-mode has no on-demand build). Prod restart must be followed by a build sweep, or there is a regression window.
