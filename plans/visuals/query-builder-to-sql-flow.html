Data-shape transformations  ·  layered

StageShapeOwner
1 URL artifact ?query={"measures":["recharge.revenue_vnd"],"timeDimensions":[…]} QueryBuilder deep-link
2 Cube Query JSON { measures, dimensions, timeDimensions, filters, segments, order, limit } Cube SDK (@cubejs-client/core)
3 HTTP request POST /cube-api/v1/load + x-cube-workspace + x-cube-game + Authorization HttpTransport · QueryBuilder.tsx:134
4 Backend forward POST {cubeCtx.cubeApiUrl}/load with minted-or-passthrough Bearer cube-proxy.ts:forward()
5 Cube compile YAML cubes + join graph → Trino-dialect SQL Cube.js server (or cube-api Docker wrapping it)
6 Trino exec SQL → rows from game_integration.<schema>.<table> Trino → lakehouse
7 Result envelope { data: Row[], annotation, query, sql? } Cube → bubbles back unchanged

Workspace flavours  ·  diverge only at step 4

local minted JWT, per-game schema

Server signs an HS256 JWT carrying { game_id }. Cube uses repositoryFactory to pick the per-game schema.

cubeApiUrlhttp://localhost:4000 authModeminted gameModelgame_id FE filternone — Cube already scopes by schema

prod (local mirror) open, flat prefix namespace

Open /load, no token. Cube serves a flat namespace of prefix-named cubes (ballistar_*, cfm_*, cros_*). FE filters which to show by gamePrefixMap[activeGameId].
