## Routes Implemented

### `debug.ts` — 4 GET routes

| Route | Auth | Scope | Status codes |
|---|---|---|---|
| `GET /debug/sessions` | X-Owner-Id required → 401 | `listSessionsForDebug` filters by `owner_id`; no archived exclusion | 200 |
| `GET /debug/sessions/:id` | X-Owner-Id required → 401 | `session.owner_id !== ownerId` → 403; missing → 404 | 200 / 401 / 403 / 404 |
| `GET /debug/turns/:turnId` | X-Owner-Id required → 401 | JOIN `chat_sessions.owner_id` via `getTurnOwnerId()` → 403/404 | 200 / 401 / 403 / 404 |
| `GET /debug/turns/:turnId/raw` | X-Owner-Id required → 401 | Same `getTurnOwnerId()` owner check | 200 / 401 / 403 / 404 |

Turn owner check uses single JOIN query:
```sql
SELECT cs.owner_id FROM chat_sessions cs
JOIN chat_turns ct ON ct.session_id = cs.id
WHERE ct.id = ?
```
Returns null → 404; mismatch → 403.

### Pagination (`/raw`)
- Default limit: 200, max: 1000
- Cursor: exclusive lower bound on `id` (autoincrement), passed as `?cursor=<n>`
- `nextCursor` = last row's `id` or `null` when returned rows < limit

### `legacy` flag
Computed per-turn: `llmCallCount === 0 && toolInvocationCount === 0 && sdkCount === 0`
Uses `countObservabilityRowsByTurn()` which does 3 `SELECT COUNT(*)` queries per turn (returns 0 on empty tables — safe for legacy installs).
