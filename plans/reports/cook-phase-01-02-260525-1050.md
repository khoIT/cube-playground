## SDK Type Investigation Finding

**Per-message usage: NO.**

Evidence from `chat-service/src/core/sse-stream.ts:42-67` (minimal structural types that mirror the actual SDK):
- `SdkAssistantMessage.message` only exposes `content: SdkContentBlock[]` — no `usage` field
- `usage` is only present on `SdkResultMessage` (the final message): `{ input_tokens?, output_tokens? }` and `total_cost_usd?`

Per-call token counts are therefore unavailable. The observer emits `inputTokens: 0, outputTokens: 0` on each `onLlmCall`. Aggregate totals are captured from the `result` SseEvent in turn.ts and persisted to `chat_turns.input_tokens / output_tokens`. This is documented in `observer-types.ts` and `sdk-event-extractor.ts` via code comments.
