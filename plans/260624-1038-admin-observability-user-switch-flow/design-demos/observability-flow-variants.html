Observability · easier user switching + a real way back

Two flow proposals for /admin/observability and the per-user drill-in. Same warm-cream design system, tokens, and header pattern as the rest of the hub — only the navigation IA changes.

Variant A

Inline switcher + breadcrumb

Recommended — lowest-risk

Keeps today's page structure. Adds a searchable user switcher + prev/next steppers to the per-user header, a breadcrumb that always points back to the real roster (Users & Access), and makes the heavy LLM-auth + Cost blocks collapsible (collapsed by default) on the org overview. Solves all three with no layout rewrite.

Variant B

Master–detail roster rail

Observability becomes a two-pane surface: a persistent searchable user roster on the left, the selected user's profile on the right. Switching = click a row; the list is always present so "back to list" is never a dead-end. With no user selected, the right pane shows the org overview (KPIs + collapsible LLM/Cost). Best if you switch users often — bigger structural change.
