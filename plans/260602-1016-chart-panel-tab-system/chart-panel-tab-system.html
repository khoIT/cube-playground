Chart panel · 2-layer tab system

Three asks, one root problem: the panel stacks two levels of selection — mode (Chart / Analysis / Compare) over view (Line / Bar / Area / Table). When both levels shout in the same orange, the hierarchy blurs. Below: the diagnosis, a color rule, and four buildable directions. Pick one, or mix.

The color system for 2 layers (answers ask #3)

One principle resolves the whole panel: only one saturated brand fill per visual group. The two layers must use different treatments so the eye reads depth, not a tie.

Layer 1 Mode — neutral lift

  • Chart / Analysis / Compare = the outer, slower-changing choice.
  • Active = white card + soft shadow + brand-tinted icon. No fill.
  • Brand shows only in the 15px icon — a whisper, not a shout.
  • Tokens: --bg-card · --shadow-pane · icon --brand.

Layer 2 View — brand fill

  • Line / Bar / Area / Table = the inner, frequently-toggled choice.
  • Active = solid --brand fill + white text. The loud one.
  • Lives in a recessed --bg-muted track → reads as nested.
  • Same treatment reused for Analysis's Breakdown/Distribution/Funnel.

Rule of thumb: outer layer = tone & elevation (neutral) · inner layer = hue (orange). Never two orange fills competing in one frame. This is how Linear, Vercel & Stripe nest their tab levels.

Directions

V1

Refined two-row — lowest risk

Keep both rows; just fix the metrics. Segments grow to 30px to match the action buttons, a divider separates the view-toggle group from the actions, and the orange-fill / neutral-track hierarchy is preserved. Smallest diff to ship.

Pros
  • Button heights consistent (ask #1 ✓)
  • Familiar — no muscle-memory reset
  • Pin shrinks to an icon to win back space
Cons
  • Still two rows (~80px) — doesn't fully answer ask #2
V2

Single merged row — "no separate line" (ask #2)

Collapse to one line. L1 becomes understated underline tabs (left); the view-toggle becomes a compact icon-only segmented control (right), with Code + a kebab overflow for Pivot/Pin. Saves a full row. Trade-off: chart-type labels go to tooltips.

Pros
  • One row — most compact (ask #2 ✓)
  • Underline L1 is quietest → L2 fill clearly dominant
  • Reads like a modern editor toolbar
Cons
  • Icon-only chart types need clear glyphs + tooltips
  • Pivot/Pin hidden behind overflow
  • Tight on narrow panes
V3

Segmented L1 + recessed L2 — clearest hierarchy

L1 becomes a proper segmented pill (active = white card lift inside a muted track). L2 sits one level deeper in its own recessed track. Depth + tone do the talking; this is the textbook answer to ask #3 and scales perfectly to the Analysis mode below.

  • The muted track on muted track needs the border to separate

Analysis sub-tabs → orange (ask #2, simplicity)

You liked the Analysis tab's simplicity but the Breakdown / Distribution / Funnel buttons are AntD blue. Replace the AntD Radio.Group with the same L2 segmented control used for chart types — instant system consistency, zero new patterns.

Recommendation

Ship V1 now (fixes the height + alignment with a tiny diff), and adopt V3's color discipline immediately for the Analysis sub-tabs (orange segmented, kill the AntD blue). If vertical space later gets tight on small screens, V2's merged row is the natural next step. V3's segmented-pill L1 is the most "designed" end-state — worth it if you want the panel to feel first-class, but it's the largest change so treat it as a follow-up.
